# AGENTS Notes

- 本机可直接使用 `scoop` 命令（PowerShell 环境已可用）。
- 每次修改完成后都要同步到 GitHub（至少提交并推送对应分支）。
- 每次修改（无论小更新还是大更新）都必须更新应用版本号，并在应用内版本信息/更新日志中体现。
- 只有大更新才发布 GitHub Release；小更新不发布 GitHub Release。
- 每次准备提交或发布前，必须先在本地完成打包验证（至少先执行 Windows 打包），打包通过后再进行后续 GitHub 同步与发布动作。
- 如任务需要额外包/工具，允许 Codex 自行安装（优先使用 `scoop` 或官方可靠渠道）。
